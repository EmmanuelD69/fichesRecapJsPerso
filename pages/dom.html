  • DOM / DOCUMENT OBJET MODEL

    On peut schématiser le DOM comme un arbre généalogique que l'on appel l'objet "window" ou l' objet global.
    Il contient un ensemble de paramètres représentant les éléments contenus dans une page web
    et dont l'objet "document" est le point d'origine / la racine.
    Pour y accéder il suffit de taper dans la console "window.document" ou "document".

    REPRESENTATION GRAPHIQUE DU DOM

  • CIBLER UN ELEMENT DU DOM

    AVEC "GETELEMENTBY" qui va retourner une "HTMLCollection" dans la console.

    1 - Selectionner un ou plusieurs éléments par leur nom de balise:

    const headers = document. getElementsByTagName( "h2");

    2 - Sélectionner un ou plusieurs éléments par leur nom de classe:

    const list = document. getElementsByClassName( "paragraphe");

    3 - Sélectionner un élément par son ID:

    const section1 = document. getElementById( "liste");


    AVEC "QUERYSELECTOR" qui va retourner une "NodeList" dans la console.
    On utilise les selecteurs CSS pour identifier ce que l'on veux cibler: (.classe) / (#Id) / (balise)

    1 - Sélectionner un élément par son nom de balise:
    !!! Si plusieurs éléments h2 existent, celui qui s'affichera dans la console sera le premier rencontré en partant du haut vers le bas. !!!

    const header = document. querySelector( "h2");

    2 - Selectionner tous les éléments portant le même nom de balise:

    const headers = document. querySelectorAll( "h2");

    3 - Sélectionner un éléments par son nom de classe:
    !!! Si plusieurs éléments portent le même nom de classe, celui qui s'affichera dans la console sera le premier rencontré en partant du haut vers le bas. !!!

    const liste = document. querySelector( ".paragraphe");

    4 - Selectionner tous les éléments portant le même nom de classe:

    const listeAll = document. querySelectorAll( ".paragraphe");

    5 - Sélectionner un élément par son ID:

    const section1 = document. querySelector( "#intro");

    PS: LA METHODE QUERYSELECTOR EST UN PEU PLUS LENTE A S'EXECUTER QUE GETELEMENTBY.

    Quelques exemples:
    const header3 = document. querySelector( "h2.item");
    dans cet example, on cible une balise h2 qui porte le nom de class "item".

    const lienSite1 = document. querySelector( "#liste a");
    dans cet example, on cible un lien se trouvant dans un élément ayant pour ID "liste".

  • HTMLCollection Vs NodeList

    HTMLCollection:

    const collectionArticles = document. getElementsByClassName( "article");
    Sélection de tous les éléments ayant pour classe "article".

    Si l'on effectue un console log, cela nous affichera une "HTMLcollection" contenant l'ensemble des éléments ayant pour classe "article".

    HTMLCollection { 0: li.article, 1: li.article, 2: li.article, 3: li.article, 4: li.article, 5: li.article, 6: li.article, length: 7 }

    "li.article" est aussi appelé un node de type élément. Il s'agit des balises HTML.
    HTMLCollection n'affiche que des nodes de types élément, il ne peut pas en afficher d'autres.

    Si l'on souhaite modifier le contenu d'une balise HTML avec Javascript, il faut le faire via une HTMLCollection

    NodeList:

    const nodeArticles = document. querySelectorAll( ".article");
    Sélection de tous les éléments ayant pour classe "article".

    Si l'on effectue un console log, cela nous affichera une "NodeList" contenant l'ensemble des éléments ayant pour classe "article".

    NodeList(7) [ li.article, li.article, li.article, li.article, li.article, li.article, li.article ]

    NodeList peut afficher différents types de nodes: élément node, text node, comment node, etc..

    Example:
    const nodeArticles = document. querySelector( ".liste-p");
    console.log(nodeArticles.childNodes);
    childNodes demande à afficher les nodes enfant de l'élément ciblé(nodeArticles).

    NodeList(27) [ #text, li.article, #text, li.article, #text, li.article, #text, li.article, #text, li.article, … ]

    !!! POUR ACCEDER UNIQUEMENT AUX "NODES ELEMENTS" DEPUIS UNE NODELIST, ON UTILISE LA METHODE "CHILDREN" SUR L'ELEMENT CIBLE !!!

    Example:
    console.log(nodeArticles.children);

    HTMLCollection { 0: li.article, 1: li.article, 2: li.article, 3: li.article, 4: li.article, 5: li.article, 6: li.article, 7: li, 8: li, 9: li, … }

    ON PEUT DONC UTILISER FACILEMENT QUERYSELECTOR POUR AFFICHER UNE HTMLCOLLECTION

  • AJOUT D'UN NODE ELEMENT

    Soit une liste d'articles:
    <ul id="liste-articles">
     <li class="article">article 1</li>
     <li class="article">article 2</li>
     <li class="article">article 3</li>
    </ul>
    <h3 class="articles-count"><b></b></h3>

    Avec HTMLCollection - (getElementById)

    1- Ciblage des nodes HTML "li" contenant la classe "article":
    const collectionArticles = document.getElementsByClassName("article");

    2- Ciblage du Node HTML "ul" contenant la classe "liste-articles":
    const listArticles = document.getElementById("liste-articles");

    3- Ciblage du Node HTML "h3" qui va nous permettre d'afficher le nombre total d'articles:
    const articlesCount = document.getElementsByClassName('articles-count b')[0];
    le [0] spécifie le node HTML sur lequel on veut agir, car par défaut getElementsByClassName va nous afficher une HTMLCollection et ici h3 est le seul et unique élément situé en position 0 de celle ci.

    4- Création d'un Node HTML "li":
    const newArticle = document.createElement("li");

    5- Ajout de la classe "article" au Node HTML "li" nouvellement créé:
    newArticle.classList.add("article");

    6- Ajout d'un texte à l'article nouvellement créé:
    newArticle.innerText = "article 4.";

    7- Ajout du nouvel article à la liste d'article:
    listArticles.appendChild(newArticle);

    8- Affichage du total d'articles:
    articlesCount.innerText = collectionArticles.length;

    Avec NodeList - (querySelector)

    1- Ciblage de l'élément Node "Parent" contenant la liste des articles:
    const listArticles = document.querySelector("#liste-articles");

    2- On récupère la liste des éléments Node "enfants" à partir du parent:
    const collectionArticles = listeArticles.children;
    L'utilisation de "children" sur l'élément Node "parent" permet d'obtenir une HTMLCollection.

    3- Ciblage du Node HTML "h3" qui va nous permettre d'afficher le nombre total d'articles:
    const articlesCount = document.querySelector('.articles-count b');

    4- Création d'un Node HTML "li":
    const newArticle = document.createElement("li");

    5- Ajout de la classe "article" au Node HTML "li" nouvellement créé:
    newArticle.classList.add("article");

    6- Ajout d'un texte à l'article nouvellement créé:
    newArticle.innerText = "article 4.";

    7- Ajout du nouvel article à la liste d'article:
    listArticles.appendChild(newArticle);

    8- Affichage du total d'articles:
    articlesCount.innerText = collectionArticles.length;

  • INTRODUCTION AUX EVENT LISTENERS

    Les "events" sont comme des capteurs qui attendent patiemment qu'un action précise se produise sur un élément déterminé. Une fois cette action détectée elle déclenche une réaction de notre choix.

    Ces évènements peuvent être de plusieurs types comme:
    -CLICK : un clic sur un bouton par example,
    -SCROLL : utiliser le bouton centrale de la souris pour scroll down/up,
    -RESIZING BROWSER : changer la dimension de la fenêtre du navigateur,
    -HOVERING : survoler un élément ou une zone de la page web,
    pour n'en citer que quelques uns...

  • INSTALLER UN EVENT LISTENER (un capteur)

    C'est une procédure qui s'effectue en 2 temps:

    1-CIBLER L'ELEMENT A SURVEILLER:
    const (ou let) nom-de-la-variable = document.querySelector("classe ou id de l'élément ciblé")

    Si plusieurs éléments portant le même nom de classe doivent être sélectionner, on utilisera la méthode "querySelectorAll"

    Example:
    const button = document.querySelector("#submit");

    2-INSTALLER L'EVENT LISTENER AVEC "ADDEVENTLISTENER":
    (cible).addEventListener("event", function anonyme ou pas() {
     action(s) à effectuer...
    });

    Example, Si je clic sur le bouton, un message s'affichera dans la console:
    button.addEventListener("click", function() {
     console.log("coucou c'est moi");
    });

    3-SIMPLIFICATION / FACTORISATION:
    const button = document.querySelector("#submit").addEventListener("click", function() {
     console.log("coucou c'est moi");
    });

    ou

    const button = document
     .querySelector("#submit")
     .addEventListener("click", function() {
      console.log("coucou c'est moi");
    });

  • FUNCTION(EVENT) à quoi sert "EVENT"?

    lorsqu'on souhaite utiliser certains évènements déclencheurs comme l'utilisation d'une touche du clavier, ou un scroll up/down, etc...
    function(event) ou function(e) va nous permettre de passer en paramètre de la fonction l'information indiquant sur quoi l'utilisateur est en train d'agir.
    Pour y parvenir, il faut indiquer à notre fonction quel est le type d'évènement déclencheur(click/keydown/scroll/etc...).
    Pour consulter les différents types d'events qui existent:
    liste des évènements déclencheurs

    Partons du principe que l'on crée un bouton sur lequel on install un eventlistener:
    button.addEventListener("keydown", function (event) {
     console.log(event);
    });

    Quand on appuiera sur le bouton cela va déclencher le capteur "keydown" qui va detecter quelle touche du clavier est préssée par l'utilisateur et la fonction nous affichera les détails de celle ci dans la console grâce à la propriété "event" placé en paramètre de la fonction anonyme.

    résultat en console:
    keydown { target: button#submit , key: "s", charCode: 0, keyCode: 83 }
    keydown { target: button#submit , key: "a", charCode: 0, keyCode: 65 }
    keydown { target: button#submit , key: "l", charCode: 0, keyCode: 76 }
    keydown { target: button#submit , key: "u", charCode: 0, keyCode: 85 }
    keydown { target: button#submit, key: "t", charCode: 0, keyCode: 84 }

    Ainsi, en utilisant les informations obtenues sur le "keycode", on peut facilement décider de la touche qui va être le déclencheur de l'action.
    En fonction que cette touche soit préssé ou non (true ou false), alors on lance notre code.

    button.addEventListener("keydown", function (event) {
     if (event.keyCode === 65) { si le keycode est égal à 65
      title.classList.toggle("nouvelArticle"); changer la classe du h3
     }
    });

    !!! Dans le cas présent il faut presser le bouton 1 fois pour que que l'event soit actif !!!

  • AJOUTER UN EVENT LISTENER A UN ENSEMBLE D'ELEMENTS

    Prenons l'exemple de notre liste d'articles, et imaginons qu'après avoir créer plusieurs articles, nous souhaitons en supprimer certains. Pour que cela puisse être possible il nous faut alors ajouter un eventListener sur chacun des articles et indiquer que l'action qui sera effectué lorsque l'on click sur l'article est une suppression(delete).

    Tout d'abord, un exemple simple:
    Nous avons déjà quelques articles présents, et nous souhaitons implémenter l'option de suppréssion d'articles.
    On va dans un premier temps, "looper" sur la liste d'articles déjà existante afin d'ajouter à chaque article un eventListener en précisant que le déclencheur d'action sera un "click" et que l'action sera d'afficher un message(delete).

    const listeArticles = document.querySelector("#liste-articles"); ciblage de l'ul contenant les articles
    const articles = listeArticles.children; affichage de la liste des articles dans une HTMLCollection.

    for(article of articles){Pour chaque article de la liste d'articles.
     article.addEventListener("click", deleteArticle); ajoute un eventListener de type "click" qui déclenchera l'action "deleteArticle".
    }

    il faut définir l'action "deleteArticle" qui est une fonction:
    function deleteArticle() {
     console.log("delete article"); affiche dans la console le message "delete article".
    }

    A chaque fois que l'on va click sur un article, le message "delete article" apparaitra dans la console. On peut donc envisager de supprimer un article en appuyant dessus, au lieu d'écrire un message en console.

    Cette Example fonctionne dans le cas où une liste d'article existe déjà et nécéssite l'ajout d'une option "delete".
    Si l'on souhaite ajouter de nouveau articles et y greffer l'option de suppression, il va falloir utiliser une autre méthode.

  • AJOUTER DE NOUVEAUX ELEMENTS AVEC EVENT LISTENERS

    Partons du principe que nous n'avons pas d'article sur notre page, et que nous souhaitons pouvoir en ajouter, mais aussi les supprimer à notre convenance.
    Nous savons déjà comment faire pour ajouter un nouvel élément.Il nous suffit d'ajouter un eventListener sur chaque nouvel article pour pouvoir intéragir dessus à notre souhait.
    Dans le cas présent, nous souhaitons utiliser l'eventListener de sorte que lorsque l'on clic sur un article cela déclenchera sa suppréssion(deleteArticle).

    button.addEventListener("click", function () {ajout d'un eventListener qui déclenche une série d'actions au click.
     const newArticle = document.createElement("li"); création d'une li
     newArticle.classList.add("article"); ajout d'une classe
     newArticle.innerText = `cet article est le n° ${articles.length + 1}`; ajout d'un texte
     listeArticles.appendChild(newArticle); ajout de la "li" et tout ce qu'elle contient dans la liste
     articlesCount.innerText = articles.length; Affichage du nombre d'article dans la liste.
     newArticle.addEventListener("click", deleteArticle); ajout d'un eventlistener sur chaque nouvel article avec pour évènement déclencheur un "click" et pour action la suppression de l'article(deleteArticle).
    });

    Bien évidemment, cela ne suffit pas, il faut aussi définir la fonction qui va permettre d'effacer l'article.
    On viens juste de placer un eventListener qui pour le moment ne fais rien tant que la fonction "deleteArticle" n'existe pas.

    logique derrière la fonction:
    Nous savons que lorsque l'on click sur un élément qui possède un eventListener, des informations sont collectées et rendues disponibles grâce au paramètre "event" si celui ci est intégré à la fonction.
    Parmis ces informations se trouve l'identifiant unique de l'élément qui a été activé, on peut le trouver sous la propriété ayant pour key "target".
    Pour pouvoir visualiser la valeur de "target", il nous faut d'abord créer une fonction qui aura pour paramètre l'évènement(e ou event).
    Il ne reste plus qu'a afficher en console le contenu => console.log(e);
    Dans notre cas, la fonction que l'on cherche à créer est la fonction supprimer un article(deleteArticle).

    function deleteArticle(e) { définition d'une fonction que l'on appel "deleteArticle" et pour laquelle on passe en paramètre l'évènement(e ou event ou click ou hover etc...).
     console.log(e); on affiche en console les information sur l'évènement
    }

    on obtient, en console, l'objet suivant en cliquant sur l'article fraichement crée:
    MouseEvent {isTrusted: true, screenX: 116, screenY: 502, clientX: 116, clientY: 399, …}

    Ce qui nous donne un tas d'infos, dont l'identifiant unique de l'élément ciblé(target).
    on peut utiliser cette information pour localiser et supprimer l'élément lorsqu'on clic dessus.
    Pour cela il suffit simplement de rajouter à notre fonction cette commande.

    function deleteArticle(click) {
     click.target.remove(); supprime l'élément cible du click.
    }

  • EVENT BUBBLING - SUPERPOSITION D'EVENEMENTS

    Nous avons vu qu'il est possible de placer des eventListeners sur autant d'éléments que nous le souhaitons. Cependant, il faut garder à l'esprit qu'un click peut déclencher plusieurs actions à la fois!

    Par example si une "ul" avec un "eventListener" contient plusieurs "li" avec elles aussi des "eventListener", lorsque l'on va cliquer sur l'une des "li" cela va déclencher à la fois l'action de la "ul" et celle de la "li".
    Il va donc y avoir conflit, et on veux faire en sorte que cela n'arrive pas.

    Il existe pour cela une solution très simple qui consiste à ajouter dans la fonction la commande suivant:
    click.stopPropagation();

    function deleteArticle(click) {
     click.stopPropagation();
     click.target.remove();
    }

  • EVENT.PREVENTDEFAULT() - EMPECHER LE FONCTIONNEMENT NORMAL D'UN EVENT

    Annule l'action par défaut déclenché par un événement si et seulement si la propriété "cancelable" de l'event vaut true, sinon n'a aucun effet.

    Le comportement par défaut peut être par exemple :

    - l'affichage d'une erreur JavaScript dans la console.
    - la non prise en compte d'une frappe de clavier si le caractère n'est pas autorisé.
    - le renvoi vers une URL lorsqu'on clic sur un lien.
    - la soumission d'un formulaire sur l'évènement submit.

    EXEMPLE:
    ---DECLARATION D'UNE CONSTANTE POUR CIBLER LE BOUTTON SUBMIT---
    C'est un bouton qui se trouve dans un formulaire et qui a pour action lorsqu'on click dessus de collecter et envoyer les informations saisies dans le formulaire puis de "raffraichir" la page en la remettant à son état originel

    const button = document.querySelector("#submit");

    ---AJOUT D'UN ARTICLE---
    Ici, nous allons empécher l'action par défaut du bouton "submit" pour la remplacer par l'ajout d'un article.

    button.addEventListener("click", function (e) {
     e.preventDefault();
     const newArticle = document.createElement("li");
     newArticle.classList.add("article");
     newArticle.innerText = `cet article est le n° ${articles.length + 1}`;
     listeArticles.appendChild(newArticle);
     articlesCount.innerText = articles.length;
     newArticle.addEventListener("click", deleteArticle);
    });

    Au "click" sure le bouton on empêche immédiatement l'action par default de celui ci qui est de refresh la page en envoyant les données collectées. Si nous n'avions pas interrompu l'action par defaut cela aurait eu pour effet d'effacer l'article créé et de revenir aux formulaire vide par defaut.
